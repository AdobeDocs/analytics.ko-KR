# Report Builder 문제 해결 및 우수 사례

Report Builder 전달을 최적화하는 방법 및 때때로 발생할 수 있는 오류 메시지 목록입니다.

## Report Builder 5.0 사용자 및 Opening 5.1 통합 문서 {#section_C29898775999453FABB5FB0E098415C8}

Adobe는 차원과 분류 사이의 구분 기호를 밑줄 문자(_)에서||. 이 변경 사항은 분류 요청으로 Report Builder v5.1 통합 문서를 여는 Report Builder 5.0 사용자에게 호환성을 의미합니다. v5.1 이전 버전의 통합 문서를 열 때마다 직렬화된 모든 분류 요청이 이 형식으로 변환됩니다.

이를 통해 향후 호환성 문제가 발생합니다.v5.1로 변환한 후, 통합 문서가 Report Builder v5.0에서 사용자와 공유된 경우, 해당 사용자는 분류 요청을 인식할 수 없습니다(예: &quot;_&quot;를 찾지만 v5.1은 &quot;||&quot;).

분류 요청을 포함한 ARB v5.1 통합 문서를 열면 다음의 부작용을 경험하게 됩니다.

* 통합 문서를 열면 다음 경고가 표시됩니다. &quot;이 통합 문서는 Report Builder v5.1에서 마지막으로 저장되었습니다. 이 버전에서는 이 컴퓨터에 설치된 Report Builder 버전과 호환되지 않는 기능이 몇 가지 도입되었습니다. 이 통합 문서를 업데이트하기 전에 최신 Report Builder 버전으로 업그레이드하는 것이 좋습니다.&quot;
* 분류와 함께 ARB 요청을 마우스 오른쪽 단추로 클릭하면 리포트 빌더 컨텍스트 메뉴(요청 편집, 종속 요청 추가...)가 표시되지 않습니다.
* 세 번째 단추를 클릭하거나 요청 관리자 양식에서 요청 세트를 새로 고쳐 모두 새로 고침을 수행하는 경우 분류 요청이 오류 없이 실행됩니다. 하지만 분류 값은 작성되지 않습니다.
* 요청 관리자를 열고 행에서 행으로 이동하여 올바른 요청에 도달할 때까지 요청을 편집할 수 있습니다.
* 요청을 편집하고 모든 매개 변수를 동일하게 둔 다음 마침을 클릭하면 응답이 올바르게 작성됩니다. 실제로 요청을 편집하면 응답 레이아웃 매개 변수가 다시 직렬화됨에 따라 문제가 해결됩니다. 따라서 시간이 많이 걸리더라도 해결 방법이 있습니다.

## Report Builder에서의 인증 문제 {#section_FD79104DF1414FE2B36591606C963DE6}

Report Builder에서는 보고서 세트에서 데이터 요청을 만들려면 인증이 필요합니다. [!DNL Analytics] 내 또는 네트워크에 대한 설정에 따라 Report Builder에서 로그인할 때 문제가 발생하기도 합니다.

**잘못된 로그인 회사**

이 오류는 로그인 회사를 잘못 입력한 경우나 네트워크 활동 문제가 있는 경우에 가장 일반적으로 발생합니다. 다음을 수행합니다.

* 오타나 잘못된 공백이 없는지 로그인 회사 철자를 확인합니다.
* 동일한 로그인 회사로 Analytics에 로그인하여 정확한지 확인합니다. 해당 자격 증명으로 로그인할 수 없는 경우 회사 관리자에게 연락하여 정확한 로그인 회사 정보를 받습니다.

**방화벽**

리포트 빌더는 포트 80 및 443을 사용합니다. 이러한 포트가 조직의 방화벽을 통해 허용되는지 확인합니다. 추가 방화벽 제외에 대해서는 Adobe의 내부 IP 주소를 참조하십시오.

## 요청 최적화를 위한 권장 사항 {#section_33EF919255BF46CD97105D8ACB43573F}

다음 요소는 요청을 더욱 복잡하게 만들고 처리 속도를 저하할 수 있습니다.

**전달 속도를 저하할 수 있는 요소**

* 너무 많은 책갈피, 대시보드 및 Report Builder 통합 문서가 몇 시간 내에 예약되었습니다.
* 너무 많은 리포트 빌더 통합 문서가 거의 동시에 예약되었습니다. 이러한 경우 보고서 API 큐가 백로그됩니다.

**통합 문서 런타임 속도를 저하할 수 있는 요소**

* 분류가 크게 증가합니다.
* 시간에 따라 요청 날짜 범위를 늘립니다.

   **예**: 일 세부기간으로 분류된 올해 설정을 사용하여 트렌드 요청을 만든다고 가정합니다. 올해 말에, 이 요청은 올해 초에 작성된 기간보다 더 긴 기간을 반환합니다.

   `(January 1 - January 30 versus January 1 - December 31.)`

**통합 문서 전달 실패의 원인이 됩니다.**

통합 문서의 복잡한 Excel 공식, 특히 날짜 및 시간이 포함된 공식입니다.

**0(값 없음)을 반환하는 셀**

Excel 시트 이름의 아포스트로피 또는 작은 따옴표로 인해 리포트 빌더가 값을 반환하지 않습니다. (Microsoft Excel 제한 사항입니다.)

**개별 요청 성능**

처리 속도는 다음 설정의 영향을 받을 수 있습니다.

| 설정 | 더욱 빨라진 성능 | 성능 저하 |
|--- |--- |--- |
| 분류 및 분류 순서 | 소수 | 많이 |
|  | 예: A에서 Z까지 분류하는 경우 A에 대한 항목 수는 Z에 대한 항목 수보다 항상 적어야 합니다. 반대의 경우 요청 시간이 많이 늘어날 수 있습니다. |
| 날짜 범위 | 작은 범위 | 광범위 |
| 필터링 | 특정 필터링 | 가장 빈도가 높은 필터링 |
| 세부기간 | 집계됨 | 시간별<ul><li>일별</li><li>주별</li><li>월별</li><li>분기별</li><li>연간</li></ul> |
| 항목 수 | 작은 데이터 세트 | 대용량 데이터 세트 |


**예약 시간**

24시간 동안 시차를 두고 예약하십시오(아래 표 참조). 서로 가깝게 예약된 기존 책갈피, 대시보드 및 리포트 빌더 통합 문서는 지연을 발생시킬 수 있습니다.

더 크고 복잡한 요청을 오전 일찍 예약하면 업무일 동안 수동으로 가져오기 및 새로 고침이 발생할 수 있습니다.

| 예약 시간 | 오전 1시오전 2시 | 오전 2시오전 7시 | 오전 7시- 오후 6시 | 오후 6시- 자정 |
|--- |--- |--- |--- |--- |
| 리포트 빌더 사용 | 조용한 | 매우 사용 중 | 클라이언트측 사용.<br>로컬에서 새로 고치고 &quot;즉시 전송&quot;을 요청하는 사용자 수가 더 많음.<br>또한 예약된 통합 문서가 시간 초과하는 경우 API 대기열이 지워졌는지 확인하십시오. | 사용 안 함 |

**시간 초과**

예약된 보고서는 4시간 후에 시간 초과됩니다. 시스템에서 세 번 더 예약을 시도하면 오류가 발생할 수 있습니다. (일반적으로 데이터 세트가 클수록 실행 시간이 오래 걸립니다.) 이러한 시간 초과는 [!DNL Analytics] 보고 및 Report Builder에서 볼 수 있습니다.

* [!DNL Analytics]: **[!UICONTROL Favorites]** > **[!UICONTROL Scheduled Reports]**

* Report Builder: Click **[!UICONTROL Management]** in the [!UICONTROL Add-ins] tab in Excel.

## 오류 메시지 설명 {#section_3DF3A1EEDAD149CB941BEABEF948A4A5}

Report Builder 사용 시 간혹 발생할 수 있는 오류 메시지 목록입니다.

>[!NOTE] 이것은 선별된 오류 메시지들로서 완전한 목록은 아닙니다. 오류 해결에 대한 자세한 내용은 관리자에게 문의하십시오.

**이 기능은 열린 통합 문서에만 적용할 수 있습니다.**

Excel에 열려 있는 통합 문서(스프레드시트 문서)가 없고 리포트 빌더 도구 모음에서 아이콘 하나를 클릭하면 이 메시지가 표시됩니다. 또한 스프레드시트를 열 때까지 도구 모음이 비활성화됩니다. 그러나 도구 모음이 여전히 활성화되어 있을 때 이 오류를 일으키지 않고 온라인 도움말 아이콘을 클릭할 수 있습니다.

**를 활성화하기[!UICONTROL Request Wizard]전에 먼저 를 종료해야[!UICONTROL Request Manager]합니다.**

및 이 [!UICONTROL Request Manager] 및 [!UICONTROL Request Wizard] 가 기능적으로 연결되어 있는 동안에는 에서 수행한 작업을 완료하거나 취소하기 [!UICONTROL Request Manager] 전에 작업을 시작할 수 없습니다 [!UICONTROL Request Wizard].

**이 범위와 연관된 요청이 없습니다.**

이 오류 메시지는 스프레드시트의 셀에 요청이 없을 [!UICONTROL From Sheet] [!UICONTROL Request Manager] 때 에서 단추를 클릭하면 발생합니다.

스프레드시트에서 요청이 들어 있는 셀을 식별하려면 의 표에 나열된 개별 요청을 클릭합니다 [!UICONTROL Request Manager]. 요청이 셀과 연결된 경우 테이블에서 요청을 선택하면 셀이 강조 표시됩니다.

**선택한 범위가 유효하지 않습니다. 다른 범위를 선택하십시오.**

스프레드시트의 셀을 선택하고 이미 매핑된 요청이 있는 경우 이 오류가 발생합니다. 셀에 매핑된 요청을 삭제하거나 매핑할 다른 셀 범위를 선택하십시오.

셀을 삭제하려면 셀을 삭제(행 또는 열 제거)하기 전에 요청이 들어 있는 셀을 찾고 요청을 삭제해야 합니다.

**이 기능을 사용하기 전에 포커스가 있는 Excel 셀을 종료하십시오.**

Excel 셀에서 *편집 모드에* 있고 리포트 빌더 아이콘 중 하나를 클릭하면 이 오류 메시지가 나타납니다. Excel 셀의 편집 모드는 셀이 선택되고 커서가 셀 내부에 나타난다는 것을 의미합니다. Excel 셀에서 바로 [!UICONTROL Formula] 막대에 입력하거나 Excel의 맨 위에 입력하는 경우에도 편집 [!UICONTROL Name Box] 모드입니다.

**선택한 범위가 다른 요청의 범위와 교차합니다. 선택 내용을 변경하십시오.**

이미 셀 집합을 스프레드시트에 매핑한 경우 이 오류가 표시됩니다.

새 요청을 추가하기 전에 매핑되는 셀을 확인하는 한 가지 방법은 해당 셀을 [!UICONTROL Request Wizard] 닫고 여는 [!UICONTROL Request Manager]것입니다. 그런 다음 요청 요약 테이블에 나열된 항목을 하나씩 선택합니다. 목록에서 요청을 선택할 때마다 스프레드시트에서 요청 매핑을 포함하는 해당 셀이 강조 표시됩니다.

이것은 여러 셀을 여러 영역에 매핑하기 전에 강조 표시, 행 또는 열 정보 또는 서식 스타일로 셀을 표시하는 것을 고려해야 하는 한 가지 이유입니다.
