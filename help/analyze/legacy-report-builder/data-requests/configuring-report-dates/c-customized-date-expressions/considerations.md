# 사용자 지정 날짜 고려 사항

{{legacy-arb}}

사용자 지정 표현식을 사용하여 날짜 범위를 설정할 때의 두 가지 중요 고려 사항:

* 보고서 실행(또는 새로 고침 요청) 날짜에 따라 사용 가능한 데이터가 결정됩니다.
* 보고서의 시작 및 종료 날짜 롤오버는 보고서에서 다루는 날짜 범위에 영향을 줍니다.

데이터 가용성은 보고서의 시간대와 보고서에서 요청을 새로 고치는 날짜 모두에 민감하므로 원하는 정보를 추출하기 위해 적절한 날짜에 보고서를 실행해야 합니다. 아래 예는 이러한 두 가지 고려 사항을 모두 보여 줍니다.

집계된 세부기간을 사용하여 [!UICONTROL 페이지 보기]를 요청한다고 가정합니다. 북아메리카에서는 주말이 일요일에 시작한다. 일요일부터 토요일(예: 2008년 11월 23일부터 11월 29일까지)까지의 기간에 대해 업데이트된 보고서를 가져오려면 이전 주(29/11/23일)에 대해 일요일(11월 30일)에 보고서(새로 고침 요청)를 실행합니다.

사용자 지정된 다음 표현식 사용:

*시작:* cw-1w *끝:* cw-1d

요청에 대한 포함 [!UICONTROL 종료 날짜]이(가) 11/30인 경우 사용자 지정 식의 분석:

*시작:* cw-1w

11월 30일 일요일에 시작하는 현재 주의 일자 - 7일 = 11월 23일 일요일에 시작하는 이전 주의 일자

*받는 사람:* cw-1d

11월 30일 일요일에 시작하는 현재 주의 날짜 - 1일 = 11월 29일 토요일

사용자 지정된 표현식이 스프레드시트에 매핑되면 2008년 11월 30일 일요일을 부동 요청에 대한 포함 [!UICONTROL 종료 날짜]&#x200B;(으)로 사용하여 요청을 새로 고치십시오. 데이터는 일주일 동안의 기간을 반영합니다.

대신 표현식을 새로 고치고 11월 29일 토요일을 부동 요청에 대한 [!UICONTROL 종료 날짜]로 지정하면 데이터에 11/16주부터 11/22주까지의 주가 반영됩니다. 요청 새로 고침에 대한 참조 날짜가 하루 앞서기 때문입니다.

요청의 [!UICONTROL 종료 날짜] 포함 날짜가 11/29인 경우의 차이점은 다음과 같습니다.

*시작:* cw-1w

11월 23일 일요일에 시작하는 현재 주의 일자 - 7일 = 11월 16일 일요일에 시작하는 이전 주의 일자

*받는 사람:* cw-1d

11월 23일 일요일에 시작하는 현재 주의 날짜 - 1일 = 11월 22일 토요일

유럽과 일부 다른 국가들에서는 주말이 일요일이 아닌 월요일에 시작됩니다. 이 경우 달력을 사용자 정의하여 시작 날짜를 변경할 수 있습니다. [사용자 지정 달력](/help/analyze/legacy-report-builder/data-requests/configuring-report-dates/custom-calendar.md)을 참조하세요.
