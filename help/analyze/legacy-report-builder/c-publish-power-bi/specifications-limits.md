# 제한 사항 및 사양

{{legacy-arb}}

## Power BI 게시 제한 사항 {#section_D4BDD70B20F94A0FAE53531CA528AE42}

>[!NOTE]
>
>이 제한 사항은 &quot;Report Builder 요청을 Power BI 데이터 세트 테이블로 게시&quot; 옵션에만 적용됩니다.

* 통합 문서당 최대 100개의 Report Builder 요청을 Power BI으로 내보낼 수 있습니다.
* 101번째 요청에 도달하면 예약 프로세스에서 요청 내보내기를 중지합니다.
* Analytics 데이터의 처음 10,000개 행만 Report Builder 요청당 Power BI으로 전송됩니다. 나머지 행은 무시됩니다.

## Power BI에 게시 후 Report Builder 요청 편집 {#section_6989E74F68DD43F08D37C36B6777DB50}

>[!NOTE]
>
>이 사양은 “모든 Report Builder 요청을 Power BI 데이터 세트 테이블로 게시” 옵션과 “통합 문서에서 형식이 지정된 테이블을 모두 Power BI 데이터 세트 테이블로 게시” 옵션에 적용됩니다.

Report Builder 요청을 Power BI에 게시한 후 편집하면 문제가 발생할 수 있습니다.

* **사례 1**: Power BI에 통합 문서를 게시하고 해당 데이터를 기반으로 시각화를 만듭니다. 그런 다음 통합 문서를 변경하여 이 통합 문서가 참조하는 데이터 세트의 열 중 하나가 사라지게 합니다. 그런 다음 다시 게시합니다. 이렇게 하면 Power BI의 시각화가 중단됩니다.

  **다음은 시각화가 중단되는 방법의 예입니다.**

   1. Report Builder에서 페이지 차원 및 페이지 보기 수 지표를 사용하여 하나의 요청으로 통합 문서를 만듭니다.
   2. Power BI에 게시하도록 이 요청을 예약합니다.
   3. Power BI에서 페이지 및 페이지 보기 횟수에 대한 시각화를 생성합니다.
   4. 이제 요청에서 페이지 보기 수를 제거하여 통합 문서를 편집합니다.
   5. 업데이트된 통합 문서로 일정을 편집하고 요청을 Power BI에 다시 게시합니다.
   6. 새 통합 문서가 Power BI으로 전송되면

      1. 처음 게시할 때 생성된 기존 데이터 세트를 덮어썼는지 확인합니다.
      2. page_1 테이블이 페이지 및 방문 열로 제대로 업데이트되었는지 확인합니다.
      3. 시각화가 page_1 테이블에 더 이상 없는 페이지 보기 열을 참조하므로 시각화가 손상되었는지 확인합니다.

  **다음은 시각화가 중단되지 않는 방법의 예입니다.**

   1. Report Builder에서 페이지 차원 및 페이지 보기 수 지표를 사용하여 하나의 요청으로 통합 문서를 만듭니다.
   2. Power BI에 게시하도록 이 요청을 예약합니다.
   3. Power BI에서 페이지 및 페이지 보기 횟수에 대한 시각화를 생성합니다.
   4. 이제 Report Builder에서 통합 문서를 편집하고 페이지 및 페이지 보기 수를 유지하면서 방문 지표를 추가합니다.
   5. 업데이트된 통합 문서로 일정을 편집하고 요청을 Power BI에 다시 게시합니다.
   6. 새 통합 문서가 Power BI으로 전송되면

      1. 처음 게시할 때 생성된 기존 데이터 세트를 덮어썼는지 확인합니다.
      2. page_1 테이블이 페이지, 페이지 보기 수 및 방문 수 열로 제대로 업데이트되었는지 확인합니다.
      3. 시각화가 page_1 표에 여전히 있는 두 열을 참조하므로 계속 제대로 작동하는지 확인하십시오.

* **사례 2**: 통합 문서의 섹션을 Power BI의 대시보드에 고정한 다음 나중에 통합 문서에서 고정된 섹션(예: 차트 또는 표)을 제거합니다. 이렇게 하면 시각화가 중단됩니다.

## Power BI 보고서 이름 변경 {#section_2E7893A78B914EBFACB2B08CBD9E472E}

공백이 밑줄 문자로 대체되는 경우를 제외하고 기본적으로 통합 문서 파일 이름(.xlsx 확장명 없음)에서 이름이 채워집니다.

주의 사항

* 레이블은 행 및 열 주소로 오인될 수 있는 문자와 숫자의 조합이 될 수 없습니다. 예를 들어 A100은 워크시트에 있는 셀 주소이므로 레이블이 될 수 없습니다.
* `'#', '@', '!', '$', '^', '&', '&#42;', '`&#39;, `'~', ' '` 문자는 유효한 레이블 문자가 아닙니다. 이 문자들은 밑줄 문자로 대체됩니다.
* 잘못된 이름을 입력하면 자동 생성된 이름을 제안하는 경고 메시지가 표시됩니다. **[!UICONTROL 예]**&#x200B;를 클릭하면 이 이름이 사용됩니다. **[!UICONTROL 아니요]**&#x200B;를 클릭하면 고급 마법사 UI에서 새 이름을 입력할 수 있습니다.
