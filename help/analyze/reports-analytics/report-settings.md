# 보고서 표시 설정 및 탐색

모든 보고서가 표시되는 방식과 기본 메뉴 옵션을 간소화된 메뉴에서 해당 위치에 매핑하는 정보를 정의하는 설정입니다.

>[!IMPORTANT]
>**2023년 12월 31일**&#x200B;부로 Adobe는 Reports &amp; Analytics 및 관련 보고서와 기능에 대한 서비스를 중단할 예정입니다. 해당 시점에 Reports &amp; Analytics 및 모든 관련 보고서와 일정이 작동 중단될 예정입니다. Reports &amp; Analytics가 제공하는 보고서, 시각화 및 기반 기술은 더 이상 Adobe의 기술 표준을 충족하지 않습니다. 대부분의 Reports &amp; Analytics 기능은 Analysis Workspace 내에서 사용할 수 있습니다. 2015년 Analysis Workspace가 출시된 이후 Reports &amp; Analytics 기능이 Analysis Workspace로 이전되면서 워크플로 패리티의 한계점에 도달했습니다. 이 공지 사항은 서비스 종료 프로세스에 대해 설명합니다.

## 보고서 표시 설정 및 탐색 {#concept_09832A2CA0FF4982B1AA37C1B635220B}

**[!UICONTROL Analytics]** > **[!UICONTROL 구성 요소]** > **[!UICONTROL 모든 구성 요소]** > **[!UICONTROL 보고서 설정]**

| 요소 | 설명 |
|--- |--- |
| **일반 설정** |  |
| 관련 지표 섹션 표시 | 관련 지표는 현재 보고 있는 보고서와 밀접한 관련이 있는 지표입니다(예: 페이지 보기 보고서의 상위 5개 페이지). |
| 세부 사항 섹션에서 인터넷 평균 표시 | 수천 개의 비즈니스 웹 사이트에서 집계한 특정 통계의 평균 값을 표시합니다. 이 옵션이 활성화되면 해당 섹션이 보고서 요약 섹션과 보고서 세부 사항 섹션에 개별 열로 나타납니다. 이 기능은 기술 그룹의 트래픽 보고서와 검색 엔진, 언어 및 도메인 보고서에서만 사용합니다. |
| Adobe의 기본 메뉴 구조 표시 | 관리자가 사용자 기본 설정에 맞게 보고서 메뉴를 사용자 지정하는 관리 도구에서 설정을 무시하고, 보고서 메뉴가 기본 설정으로 돌아갑니다. |
| 보고서를 표시할 때 열 너비 강제 조정 | 미적인 면을 고려하여 보고서 열 너비를 조정합니다. 이 설정은 네 개 이상의 지표가 표시될 때 유용합니다. |
| **그래프 및 차트** |  |
| 트렌드 표시 그래프에 주말을 강조 표시 | 트렌드 보고서 그래프의 주말 날짜를 세로로 강조 표시합니다. 주말이 식별되면 트렌드 보고서를 보다 쉽게 평가할 수 있습니다. |
| 그래프와 요약에 예측 표시 | 미래에 발생할 특정 통계의 값을 예측합니다. 이 옵션이 활성화되면 보고서 요약 섹션에 해당 예측이 나타납니다. |
| 보고서에 달력 이벤트 표시 | 특정 이벤트와 관련된 사이트 실적을 추적합니다. 이 옵션이 활성화되면 보고서에 해당 이벤트가 나타납니다. |
| 플래시 그래프 사용 | 보고서에서 플래시 그래프를 활성화합니다. 플래시 그래프를 사용하면 보고서 이미지를 보다 명확하고 역동적으로 표현할 수 있지만 쉽게 복사 또는 저장할 수 없습니다. 이미지를 빠르게 복사 또는 저장하려면 이 옵션을 비활성화하십시오(이미지는 .gif 형식으로 저장됨). 이 옵션 선택을 취소하면 보고서 도구 모음에 [그래픽 복사] 단추가 나타나지 않습니다. |
| 선택한 그래프에 &quot;다른 모든&quot; 데이터 표시 | 상위 5개 이외의 다른 모든 데이터를 단일 개체로 표시합니다. (막대 차트에는 상위 5개의 웹 페이지 또는 보고서의 다른 데이터가 표시됩니다.) |
| 보고서 그래프에 &quot;없음&quot;, &quot;지정되지 않음&quot; 및 &quot;입력/책갈피 표시&quot; 데이터 표시 | 해당 값이 지정된 지표에 대한 크레딧을 수신하지 않은 지표를 보여줍니다. |
| 등급 보고서에 스파크라인 표시 | 등급 보고서의 총계 필드에 스파크라인을 표시합니다. 이것을 보고 개별 보고서를 생성하지 않고 전체 트렌드를 빠르게 파악할 수 있습니다. |
| **가속** |  |
| 보고서 가속기를 활성화하여 보고서를 더욱 빠르게 표시 | 시간 기반 알고리즘을 사용하여 최근에 요청된 보고서를 캐시하고 가장 자주 발생하는 고유 항목만 점검하여 보고서를 빠르게 배달할 수 있는 보고서 가속기를 활성화합니다. 보고서 가속기는 요청된 보고서를 15분 동안 캐시함으로써 후속 요청에 대한 보고서를 즉시 검색할 수 있습니다. 이 설정은 앞뒤로 이동하거나 보고서를 인쇄하거나 동일한 보고서에 자주 액세스하는 경우 유용합니다. 이 옵션이 비활성화되면 보고서가 요청될 때마다 시스템에서 보고서를 다시 생성합니다. |
| 대시보드 가속기를 활성화하고 사용 가능한 캐시 버전 표시 | 대시보드를 나중에 볼 수 있도록 캐시 버전을 저장하는 대시보드 가속기를 활성화합니다. 대시보드 가속기를 사용하면 대시보드 보기를 24시간 동안 캐시하여 집약적인 데이터베이스 쿼리 및 처리가 미리 수행되므로 해당 보기를 즉시 검색할 수 있습니다. 사용 가능한 캐시된 버전이 24시간 이전에 만들어진 경우 새로운 대시보드가 생성되고 새로운 캐시된 버전이 만들어집니다. 마찬가지로 대시보드 또는 대시보드에 표시되는 reportlet을 업데이트할 때마다 캐시된 새로운 버전이 만들어집니다. 캐시는 사용자를 기반으로 합니다. 공유 대시보드를 보는 다른 사용자에게는 본인이 사용한 대시보드 가속기와 대시보드 업데이트에 따라 버전이 표시됩니다. |
| 보고서 성능 개선을 통한 네트워크 가속 기능 활성화 | Adobe 인프라와 사용자 환경 간의 경로를 최적화하여 사용자 위치로의 데이터 배달을 가속화합니다. |
| 중국에서 더 빠른 사용자 환경을 위해 지역 가속 활성화 | 지역 가속기는 특정 지역 내에서 더 빠른 사용자 환경을 제공하기 위해 지역별 가속화 도메인을 사용합니다. 현재 지역 가속은 중국에서만 지원됩니다. 중국에 있지 않은 사용자가 이 기능을 활성화하면 사용자 환경이 느려집니다. 지역 가속을 활성화한 후에는 다시 로그인해야 설정이 적용됩니다. 지역 가속기를 사용하지 않으려면 이 확인란을 선택 취소하십시오. |
| **언어/통화/인코딩** |  |
| 천 단위 구분 기호 | 천 단위마다 사용할 구분자를 선택합니다(소수점 또는 쉼표). 많은 다른 국가에서 천 단위를 구분하기 위해 소수점을 사용합니다. (이 구분 문자는 통화 뿐만 아니라 시스템 전체의 모든 숫자에 적용됩니다.) |
| 보고서 세트의 기본 통화 사용 | 보고서 세트의 기본 통화 사용 여부를 지정합니다. |
| 통화 | 데이터를 변환할 통화입니다. 이 설정에서 값을 선택하면 데이터베이스에 저장된 데이터가 영향을 받지 않지만 현재 날짜의 통화 변환 비율에 따라 변환된 값으로 표시됩니다. 통화 옵션을 구성하지 않으면(기본값으로 설정) 통화 변환이 수행되지 않고 모든 값이 미국 달러(USD)로 저장되고 표시됩니다. 데이터를 처리할 때(데이터가 표시되기 전에) 통화를 변환하려면 계정 관리자에게 문의하십시오. |
| 예약된 보고서 인코딩 | 일본어 문자 인코딩의 경우 SHIFT-JIS입니다. 확장 Unix 코드용 EUC-JP로서 주로 일본어, 한국어 및 중국어 간체에 사용됩니다. |
| CSV 구분 문자 | CSV 값을 구분하는 데 사용할 문자입니다. |
