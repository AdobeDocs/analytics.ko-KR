# 속성 IQ 개요

>[!IMPORTANT]
>
>속성 IQ는 Adobe Analytics Ultimate, Prime, Select 및 Foundation SKU의 모든 고객에게 제공됩니다.

## 속성 IQ의 비즈니스 가치 {#section_E82B97114E1641A8AE911F57AEB3240A}

"고객 움직임"은 선형이 아니며 부분적으로만 예측 가능합니다. 유기적이고 유연하므로 예측할 수 없는 경우가 많습니다. 고객마다 자신의 속도로 진행하면서 뒤로 돌아오기도 하고 멈추기도 하다가 다시 시작하기도 합니다. 이런 경우 고객 여정 전반에서 마케팅 노력이 미치는 영향을 알기 어렵습니다. 또한 여러 데이터 채널을 함께 연결하여 비즈니스 질문에 응답하고자 하는 노력을 저해하여 불완전한 고객 통찰력을 얻게 됩니다.

Adobe Analytics 속성 IQ를 사용하면 첨단 인텔리전스 팀이 고객 움직임에서 발생하는 의미 있는 참여를 이해하고, 고객을 타겟 결과로 이끄는 변곡점을 지능적으로 식별하며 마케팅 이니셔티브를 효과적으로 최적화할 수 있습니다.

![](assets/attribution_iq_problem.png)

Adobe Analytics는 다음을 통해 속성을 강화합니다.

* 유료 미디어 이외의 속성 정의: 차원, 지표, 채널 또는 이벤트는 마케팅 캠페인뿐 아니라 모델(예: 내부 검색)에 적용할 수 있습니다.
* 무제한 속성 모델 비교를 사용: 원하는 수만큼 모델을 동적으로 비교합니다.
* 구현 변경 방지: 보고서 처리 시간 및 컨텍스트 인식 세션으로 고객 움직임 컨텍스트를 구축하고 런타임에 적용할 수 있습니다.
* 사용자의 속성 시나리오와 일치하는 세션을 생성합니다.
* 세그먼트별 속성 분류: 중요한 모든 세그먼트(예: 신규 및 반복 고객, 제품 X와 제품 Y, 로열티 수준 또는 CLV)에서 마케팅 채널의 실적을 쉽게 비교할 수 있습니다.
* 채널 교차 및 다중 접점 분석: 벤 다이어그램, 막대 그래프 및 트렌드 속성 결과를 사용합니다.
* 주요 마케팅 시퀀스를 시각적으로 분석: 다중 교점 플로우 및 폴아웃 시각화를 통해 시각적으로 전환된 경로를 탐색합니다.
* 계산된 지표 작성: 여러 속성 할당 방법을 사용합니다.

## 속성 IQ의 역할은 무엇입니까? {#section_63B421E9E75B4CCEBA96726CAA37D73E}

Analysis Workspace의 속성 IQ에서는 자유 형식 테이블, 시각화 및 계산된 지표에 다양한 새로운 속성 모델 유형을 추가할 수 있습니다. 모든 속성 모델에는 다음과 같은 두 가지 구성 요소가 있습니다.

* **속성 모델**(예: 첫 번째 터치, 마지막 터치, 선형 등): 이 모델은 그룹의 히트에 대한 전환 분포를 설명합니다.
* **속성 전환 확인 기간**(예: 방문, 방문자): 전환 확인 기간은 각 모델에 대해 고려되는 히트 그룹을 설명합니다.

다음 고객 움직임 예는 3번의 방문, 3번의 전환, 4개의 마케팅 채널(검색, 디스플레이, 소셜 및 이메일)에 걸친 단일 방문자의 마케팅 터치포인트를 보여 줍니다.

![](assets/attribution_example.png)

## 인스턴스 기반 속성 {#section_A81DBC3B19014CE3894131F1CF72736F}

Analysis Workspace의 속성은 모든 차원의 '인스턴스'를 활용합니다. 즉, 속성 모델이 Analytics(처리 후 규칙, VISTA 및 마케팅 채널 처리 규칙)에 전달된 값에 적용됩니다. 실제로, 이것은 속성 모델링을 위해 prop 또는 eVar(또는 다른 차원) 사이에 차이가 없음을 의미합니다. 아래의 전환 확인 기간 또는 모델 사용이 유지되도록 Prop을 설정할 수 있으며, 속성 전환 확인 기간 또는 모델이 적용된 경우 eVar에 대한 할당 및 만료 설정은 사용되지 않습니다(관리 설정에 지정됨).

## 속성 전환 확인 기간 {#section_A2782BB64171431EB370CDCD4AD8030D}

속성 전환 확인 기간은 속성 모델이 적용되는 히트 그룹입니다. Analysis Workspace에 방문과 방문자라는 두 개의 속성 전환 확인 기간 설정이 있습니다.

**방문 전환 확인 기간**

방문 전환 확인 기간은 기본적으로 30분의 비활성 상태로 구분된 활동 시퀀스입니다. 그러나 기본값을 변경하려는 경우 [보고서 처리 시간](https://marketing.adobe.com/resources/help/en_US/reference/vrs-report-time-processing.html)을 통해 [컨텍스트 인식 세션](https://marketing.adobe.com/resources/help/en_US/reference/vrs-mobile-visit-processing.html)도 지원됩니다. 사용자 지정 세션을 사용하여 가상 보고서 세트(VRS) 내의 방문 속성 전환 확인 기간을 사용하는 경우 속성 전환 확인 기간에서 해당 계산의 기반으로 사용자 지정 방문 정의를 사용합니다. 위의 예에서 각 방문은 독립적인 속성 전환 확인으로 간주됩니다.

**방문자 전환 확인 기간**

방문자 전환 확인 기간은 Workspace 패널의 보고 기간 내에 있는 방문자의 전체 히트 수와 보고 기간을 포함하는 전체 개월을 고려합니다. 예를 들어, 보고 기간 날짜 범위가 9월 15일~9월 30일인 경우, 방문자 전환 확인 날짜 범위는 9월 1일~9월 30일이 됩니다. 방문자 전환 확인 기간에 대한 자세한 내용은 [보고 기간을 벗어나 표시되는 데이터](https://helpx.adobe.com/analytics/kb/data-appearing-outside-reporting-window.html)를 참조하십시오.

**속성 전환 확인 기간 예**

속성 전환 확인 기간의 영향을 설명하기 위해 위의 예에 선형 모델(모든 터치포인트에 동일한 크레딧을 제공)을 적용해 보겠습니다.

**방문 속성 전환 확인 기간**&#x200B;을 사용할 때 각 방문에는 별도로 분산된 전환이 있습니다.

* 첫 번째 방문의 /$10은 각각 $2.50를 받는 검색, 표시, 소셜 및 이메일 간에 균등하게 분할됩니다.
* 두 번째 방문에서 검색 및 이메일은 각각 /$5 전환의 절반을 받게 되므로, 이메일 및 검색은 각각 /$2.50를 받게 됩니다.
* 마지막으로, 최종 방문 시, 이메일은 /$2 전환에 대한 모든 크레딧을 받습니다.

**방문자 속성 전환 확인 기간**&#x200B;에서는 모든 전환이 함께 고려되지만, 여러 전환이 있으므로 계산이 약간 더 복잡합니다.

* 첫 번째 /$10 전환은 검색, 표시, 소셜 및 이메일 간에 균일하게 분할됩니다.
* 두 번째 /$5 전환은 이전 방문의 이전 채널뿐만 아니라 해당 방문에 있는 채널 간에 분할됩니다.검색 = (2/6) * /$5 = /$1.67, 디스플레이 = (1/6) * /$5 = /$0.83, Social = (1/6) * /$5 = /$0.83, 이메일 = (2/6) * /$5 = /$1.67.
* 마지막으로, 마지막 전환은 방문자의 모든 채널에서 분할됩니다.검색 = (2/7) * /$2 = /$0.57, 디스플레이 = (1/7) * /$2 = /$0.29, Social = (1/7) * /$2 = /$0.29, 이메일 = (3/7) * /$2 = /$0.86.

다음은 테이블 양식의 결과에 대한 요약입니다.

| 채널 | 매출(선형/방문) | 매출(선형/방문자) |
|---|---|---|
| 검색 | /$5.00 | /$4.74 |
| 표시 | /$2.50 | /$3.62 |
| Social | /$2.50 | /$3.62 |
| 이메일 | /$7.00 | /$5.02 |
| 합계 | /$17.00 | /$17.00 |

속성 전환 확인 기간의 이 차이는 아래에 설명된 모든 속성 모델과 비슷하게 작동합니다.

## 마케팅 채널 &amp; 마케팅 채널 처리 규칙 {#section_FCBF08A9D7C94B67B7AD76E8633E7916}

첫 번째 터치 채널 및 마지막 터치 채널뿐만 아니라 첫 번째 터치 채널 세부 사항 및 마지막 터치 채널 세부 사항을 새 속성 모델에서 사용할 수 있습니다. 그러나 향후 팀의 혼란을 피하기 위해 **마케팅 채널** 및 **마케팅 채널 세부 사항**&#x200B;과 같은 두 개의 새로운 차원을 사용하는 것이 좋습니다. 이들은 정확히 동일하게 작동하지만 이름에서 '첫 번째 터치'와 '마지막 터치'를 구분해야 하는 혼돈은 없습니다. 속성 모델이 적용되지 않은 경우 마케팅 채널 및 마케팅 채널 세부 사항은 각각 마지막 터치 채널 및 마지막 터치 채널 세부 사항과 같은 결과를 제공합니다.

속성 모델을 첫 번째 터치 채널 또는 마지막 터치 채널에 적용하면 모든 속성 설정이 선택한 속성 모델에 의해 재정의됩니다. 따라서 차원의 이름이 '첫 번째 터치 채널'이면, 선형 모델을 선택한 경우 결과는 '첫 번째 터치'가 아닌 선형 속성을 반영합니다.

또한 마케팅 채널 변수는 기존의 방문(데이터 수집 프로세스 중에 적용되는 마케팅 채널 처리 규칙에 의해 정의됨)에 종속되어 있으므로 컨텍스트 인식 세션에서 사용할 수 없습니다.

## 분류 세분화 속성 {#section_F9DE21758C4643879BE05EEAE9A34E5A}

속성 모델은 모든 값과 해당 분류에 적용할 수 있습니다. 분류된 값이 해당 키로 세분화되는 경우 Analytics에는 분류된 특정 값과 연관된 키만 포함됩니다. 예를 들어, 선형 속성 모델이 'Fruits 및 'Vegetables' 값으로 분류된 'apple', 'banana', 'carrot' 값과 함께 주어진 eVar에 적용되었을 때 'Vegetables' 값이 기본 eVar로 분류되면, 'carrot'만 분류에 표시됩니다. 마찬가지로, 기본 eVar에 의해 분류된 'Fruits'는 속성 크레딧이 3개의 기본 eVar 값 모두에 분산된 경우에도 분류 결과에 'apple'과 'banana'만 표시됩니다.

이 동작은 마케팅 채널 차원이 마케팅 채널 세부 사항 차원으로 분류되는 보고서에도 적용됩니다.

## 분류 속성 {#section_B2E87C768B6B4DBFA8EB7DB5E2DF881E}

Analysis Workspace를 사용하면 다른 차원으로 모든 값을 분류하고 분류에 동일하거나 다른 속성 설정을 지정할 수 있습니다. 예를 들어 채널 차원에는 선형 속성이 적용되었을 수 있지만 캠페인으로 채널을 분류하면 캠페인 수준에서 다른 속성 모델을 지정할 수 있습니다.

이는 채널 수준(채널 전반에 균등하게 분배)에 1개의 속성 모델이 있는 팀에게 유용하지만 개별 캠페인에서 별도의 속성 모델을 사용하려는 개별 팀이 있고 해당 캠페인에 대한 합계가 채널 수준에서 할당된 것과 일치해야 합니다.

## 속성의 '없음' {#section_BC71DA030E45487AA3C3F6ED247A3C4A}

'없음' 라인 항목은 차원 값이 없는 곳에서 발생한 모든 전환을 나타내는 포괄적인 라인 항목입니다. 일반적으로 '없음' 라인 항목은 eVar 보고서 또는 지속성이 있는 다른 차원에만 존재합니다. 속성 모델이 적용되면 이전에 존재하지 않았던 '없음' 라인 항목이 표시될 수 있습니다. 이 모델은 이전에 존재하지 않았던 '없음' 라인 항목을 도입하는 속성에 속성 모델을 적용할 때 일반적으로 발생합니다.

## 다중 값 변수에 대한 속성

Analytics의 일부 차원에는 listVars, 제품 변수, 목록 속성 또는 머천다이징 eVars 등의 단일 히트에 대한 여러 값이 포함되어 있을 수 있습니다. Analysis Workspace를 사용하면 속성 IQ를 히트 레벨의 모든 유형의 변수에 적용할 수 있습니다. 단일 방문의 예제로 U자형(40/20/40) 속성 사용:

| 히트 번호 | 다중 값 변수 | 변환 이벤트 | 히트에 대한 크레딧 백분율(U자형) |
|--- |--- |---|---|
| 1 | A, B, C | - | 40% |
| 2 | D | - | 20% |
| 3 | E,F | 1 | 40% |

이 경우, A, B, C는 모두 히트 1의 동일한 시간에 설정되어 있으며, D는 히트 2에 단독으로, E 및 F는 히트 3에 설정되었습니다.

속성 IQ는 히트에 대한 모든 백분율 크레딧을 히트에 있는 모든 값에 제공합니다. 이전 예에서 A, B 및 C는 모두 40% 또는 .4 전환을 수신하고, D는 20% 또는 .2 전환을 수신하며, E 및 F는 각각 40% 또는 .4 전환을 수신합니다. 위 히트의 U자형 속성을 사용하는 보고서에서 다음 보고서를 생성합니다.

| 다중 값 변수 | 전환(U자 모양/방문) |
|--- |---|
| 애플리케이션이 사용자 지정 JavaScript 라이브러리를 내보내는 | .4 |
| B | .4 |
| C | .4 |
| D | .2 |
| E | .4 |
| F | .4 |
| 합계 | 1 |

>[!NOTE]
>기여도 모델의 히트 수준 할당으로 인해 보고서의 각 라인 항목의 합계가 포함된 히트에 속하는 총 퍼센트 크레딧을 받는 각 값으로 인해 합계와 같지 않을 수 있습니다.

## 세분화를 통한 기여도 분석

속성은 항상 세그먼테이션 전에 실행되며, 보고서 필터가 적용되기 전에 세그먼테이션이 실행됩니다. 이러한 원칙은 세그먼트가 적용된 가상 보고서 세트(VRS)에도 적용됩니다.

예를 들어 "히트 표시" 세그먼트가 적용된 VRS를 만드는 경우 속성이 켜지면 표에서 반환되는 다른 마케팅 채널을 볼 수 있습니다. 이러한 문제는 세그먼트화되지 않은 데이터세트에 속성이 실행되기 때문입니다.
