# 대상 보고서

대상 보고서는 사이트를 방문하는 사람 유형에 대한 정보를 보여줍니다.

이 페이지에서는 사용자가 분석 작업 공간 사용에 대한 기본적인 지식을 보유하고 있다고 가정합니다. Adobe [Analytics의 도구에 아직 익숙하지 않은 경우 Google Analytics 사용자를](create-report.md) 위한 분석 작업 공간에서 기본 보고서 만들기를 참조하십시오.

## 활성 사용자

활성 사용자는 이전 1일, 7일, 14일 또는 28일 동안 사이트에 대한 누적 사용자 수를 보여줍니다. Adobe는 Google Analytics에 사용된 정확한 계산을 가지고 있지 않지만 고유 방문자 수 지표를 사용하여 선택한 날짜 범위를 기반으로 사이트에 중복 제거된 사용자 수를 볼 수 있습니다.

고유 방문자의 라인 그래프를 얻으려면

1. 왼쪽의 시각화 아이콘을 클릭하고 라인 시각화를 빈 자유 형식 테이블 위의 작업 공간으로 드래그합니다.
2. 왼쪽의 구성 요소 아이콘을 클릭한 다음 고유 **방문자** 지표를 '여기에 지표 놓기'라는 레이블이 있는 작은 공간으로 드래그합니다.
3. 다른 세부기간을 원하는 경우 원하는 날짜 범위(예: 일, **주**, **월******&#x200B;등)를드래그합니다. 을 기존 날짜 차원 헤더 위에 놓습니다.

Adobe [가](/help/components/c-variables/c-metrics/metrics-unique-visitors.md) 고유 방문자를 계산하는 방법에 대한 자세한 내용은 구성 요소 사용 안내서의 고유 방문자를 참조하십시오.

## 라이프타임 값

라이프타임 값은 두 플랫폼 모두에서 추가적으로 전문적인 구현을 필요로 하는 기능입니다. 이 데이터를 얻으려면 구현 컨설턴트와 함께 작업하는 것이 좋습니다.

## 집단 분석

집단 분석은 동일한 사용자가 사이트를 다시 방문하는 빈도를 표시합니다.

집단 테이블을 생성하려면 다음을 수행합니다.

1. 왼쪽의 시각화 아이콘을 클릭하고 집단 테이블 시각화를 작업 공간으로 드래그합니다.
2. 왼쪽의 구성 요소 아이콘을 클릭한 다음 방문 **지표를** 포함 기준 및 반환 기준 모두로 드래그합니다.
3. 작성을 클릭합니다.

집단 [시각화에](/help/analyze/analysis-workspace/visualizations/cohort-table/cohort-analysis.md) 대한 추가 사용자 지정에 대한 자세한 내용은 분석 작업 공간 사용자 안내서의 집단 분석을 참조하십시오.

## 대상자

Google Analytics의 대상 보고서는 대상을 설정해야 합니다. 또한 Adobe Audience Manager를 통해 Adobe에서 구성해야 합니다. 자세한 내용은 Adobe Audience Manager 사용 안내서를 참조하십시오.

## 사용자 탐색기

사용자 탐색기 보고서를 사용하면 분석가는 익명 식별자를 통해 개별 방문을 볼 수 있습니다. Adobe는 데이터의 히트 수준 원시 내보내기인 데이터 피드 외부에 백엔드 식별자를 표시하지 않습니다.

* 분석 작업 공간에서 이 데이터를 원하는 경우 구현 컨설턴트와 협력하여 익명 고유 식별자 쿠키 값을 eVar로 전달할 수 있습니다. 이 기능은 월별 고유 방문자 수가 100만 명 미만인 소규모 구현에서만 작동합니다.
* 데이터 피드 내에서 이 데이터를 원하는 경우 연결된 열을 `visid_high` 사용하여 고유 방문자를 식별하는 가장 일반적인 `visid_low` 방법입니다. 데이터 피드에 대한 자세한 [내용은](/help/export/analytics-data-feed/c-getstarted/data-feed-overview.md) 내보내기 사용 안내서를 참조하십시오.

## 인구 통계 및 관심 분야 보고서

인구 통계 및 관심 분야 데이터는 사이트 사용자의 연령, 성별 및 관심사에 대한 정보를 제공합니다. 이 데이터는 Google이 사이트 간 추적 기능을 통해 수집합니다.

인구 통계 및 관심 자료 데이터는 Adobe에서 자동으로 수집하지 않습니다. 그러나 조직에서 이 데이터를 얻는 경우 Adobe Experience Cloud Platform 내의 기능인 고객 속성을 사용할 수 있습니다. 이를 통해 속성별 데이터 구성을 완벽하게 제어할 수 있으며 인구통계나 관심사에 국한되지 않습니다.

자세한 내용은 고객 속성 도움말을 참조하십시오.

## 지역 - 언어

지역 언어 보고서는 방문자 브라우저의 언어 설정별 사이트 트래픽을 보여줍니다.

언어 보고서를 만들려면:

1. 구성 요소 메뉴에서 언어 **차원을** 찾아 '여기에 차원 놓기'라는 큰 자유 형식 테이블 영역으로 드래그합니다.
2. 원하는 지표를 자동으로 생성된 발생 지표와 함께 작업 공간으로 **드래그합니다** . 각 [지표를 얻는 방법에 대한 자세한 내용은 지표 번역 안내서를](common-metrics.md) 참조하십시오.

자세한 [내용은 구성](/help/components/c-variables/dimensionslist/reports-languages.md) 요소 사용 안내서의 언어 차원을 참조하십시오.

## 지역 - 위치

지리적 위치 보고서는 국가별 데이터를 분류하여 전세계 맵 보기를 제공합니다.

지역 위치 보고서를 만들려면:

1. 왼쪽의 시각화 아이콘을 클릭하고 맵 시각화를 빈 자유 형식 테이블 위의 작업 공간으로 드래그합니다.
2. 왼쪽의 구성 요소 아이콘을 클릭한 다음 고유 **방문자** 지표를 '지표 추가'라는 레이블의 공간으로 드래그합니다.
3. 작성을 클릭합니다.

맵과 함께 표를 원할 경우:

1. 구성 요소 메뉴에서 국가 **차원을** 찾아 '여기에 차원 놓기'라는 큰 자유 형식 테이블 영역으로 드래그합니다.
2. 원하는 지표를 자동으로 생성된 발생 지표와 함께 작업 공간으로 **드래그합니다** . 각 [지표를 얻는 방법에 대한 자세한 내용은 지표 번역 안내서를](common-metrics.md) 참조하십시오.

자세한 [내용은 구성 요소](/help/components/c-variables/dimensionslist/reports-geosegmentation.md) 사용 안내서의 지리 특성 차원을 참조하십시오.

## 동작 - 새로운 기능과 반환

새 보고서와 재방문 보고서는 첫 번째 세션(새 방문)과 후속 세션(재방문)에 대한 간단한 보기를 제공합니다.

새 방문 수와 재방문 보고서를 만들려면 다음을 수행하십시오.

1. 구성 요소 메뉴에서 **첫 번째 방문** 세그먼트를 찾아 '여기에 차원 놓기'라는 큰 자유 형식 테이블 영역으로 드래그합니다. 처음 **방문 횟수는** 세그먼트이지만 작업 공간은 일반적으로 차원을 사용하여 행을 나타냅니다.
2. 재방문 **횟수** 세그먼트를 찾아 세그먼트 행 헤더 위로 드래그합니다. 이렇게 하면 세그먼트가 처음 방문 횟수 아래의 차원으로 추가되어 쉽게 비교할 수 있습니다.
3. 원하는 지표를 자동으로 생성된 발생 지표와 함께 작업 공간으로 **드래그합니다** . 각 [지표를 얻는 방법에 대한 자세한 내용은 지표 번역 안내서를](common-metrics.md) 참조하십시오.

라인 그래프도 필요한 경우:

1. 왼쪽의 시각화 아이콘을 클릭하고 자유 형식 테이블 위의 작업 공간으로 선 시각화를 드래그합니다
2. 자유 형식 테이블의 각 행에서 Ctrl 키(Windows) 또는 Cmd 키(Mac)를 사용하여 행을 강조 표시합니다. 이렇게 하면 두 트렌드가 라인 시각화에 나타날 수 있습니다.
3. 선 시각화의 왼쪽 위 모서리에 있는 작은 둥근 색상 점을 클릭한 다음 '선택 잠금' 확인란을 클릭합니다.

빈도 및 최근 보고서는 분석 작업 공간의 **방문** 번호 차원과 거의 같습니다.

자세한 내용은 [구성 요소](/help/components/c-variables/dimensionslist/reports-visitor-number.md) 사용자 안내서의 방문 번호 차원을 참조하십시오.

## 행동 - 참여

참여 보고서는 방문당 체류 **시간 - 버킷 차원과 거의** 같습니다.

1. 구성 요소 메뉴에서 방문당 체류 **시간 - 버킷** 차원을 찾아 '여기에 차원 놓기'라는 레이블이 있는 큰 자유 형식 테이블 영역으로 드래그합니다.
2. 원하는 지표를 자동으로 생성된 발생 지표와 함께 작업 공간으로 **드래그합니다** . 각 [지표를 얻는 방법에 대한 자세한 내용은 지표 번역 안내서를](common-metrics.md) 참조하십시오.

자세한 내용은 [구성 요소](/help/components/c-variables/dimensionslist/reports-time-spent-per-visit.md) 사용 안내서의 방문당 체류 시간 차원을 참조하십시오.

## 기술 - 브라우저 및 OS

브라우저 및 OS 보고서에서 사용할 수 있는 기본 차원은 여러 개입니다.

* 브라우저 **기본** 차원은 분석 작업 공간에서 차원으로 사용할 수도 있습니다.
* 운영 **체제** 기본 차원은 분석 작업 공간에서 차원으로 사용할 수도 있습니다.
* 화면 **해상도** 기본 차원은 분석 작업 공간에서 모니터 해상도 **차원으로 사용할 수** 있습니다.
* 화면 **색상** 기본 차원은 분석 작업 공간에서 색상 깊이 **차원으로 사용할 수** 있습니다.
* Flash **버전** 기본 차원은 Adobe Analytics에서 사용할 수 없지만 필요한 경우 eVar에서 이 데이터를 수집할 수 있습니다.

1. 구성 요소 메뉴에서 위에 언급된 원하는 차원을 찾아 '여기에 차원 놓기'라는 레이블이 있는 큰 자유 형식 테이블 영역으로 드래그합니다.
2. 원하는 지표를 자동으로 생성된 발생 지표와 함께 작업 공간으로 **드래그합니다** . 각 [지표를 얻는 방법에 대한 자세한 내용은 지표 번역 안내서를](common-metrics.md) 참조하십시오.

해당 차원에 대한 자세한 내용은 구성 요소 사용 안내서의 다음 페이지를 참조하십시오.

* [브라우저](/help/components/c-variables/dimensionslist/reports-browsers.md)
* [운영 체제](/help/components/c-variables/dimensionslist/reports-operating-system.md)
* [모니터 해상도](/help/components/c-variables/dimensionslist/reports-technology.md)
* [색상 깊이](/help/components/c-variables/dimensionslist/reports-color-depth.md)

자세한 내용은 [구성 요소](/help/components/c-variables/dimensionslist/reports-devices.md) 사용 안내서의 모바일 장치 차원을 참조하십시오.

## 사용자 지정

사용자 지정 보고서는 구현별로 정의됩니다. 조직의 Analytics 관리자 및/또는 구현 컨설턴트와 협력하여 이러한 보고서를 해석합니다. 일반적으로 조직은 솔루션 디자인 [문서를](/help/implement/prepare/solution-design.md) 유지 관리하여 사용자 지정 변수 값과 작성 방법을 추적합니다.

## 벤치마크

벤치마크 보고서를 통해 데이터의 패싯을 업계 평균과 비교하는 방법을 확인할 수 있습니다. 현재 Adobe는 고객 사이에 벤치마크 데이터를 공유하지 않습니다.

## 사용자 흐름

흐름 보고서는 두 플랫폼에서 모두 사용할 수 있습니다. 흐름 보고서를 만들려면

1. 왼쪽의 시각화 아이콘을 클릭하고 흐름 시각화를 자유 형식 테이블 위의 작업 공간으로 드래그합니다
2. 페이지 **차원을** 찾은 다음 화살표 아이콘을 클릭하여 페이지 값을 표시합니다. 차원 값은 노란색으로 표시됩니다.
3. 원하는 페이지 값을 찾아 가운데에 '차원 또는 항목'이라는 레이블이 있는 공간으로 드래그합니다
4. 이 흐름 보고서는 대화형입니다. 다음 또는 이전 페이지로 흐름을 확장하려면 값을 클릭합니다. 마우스 오른쪽 단추 클릭 메뉴를 사용하여 열을 확장하거나 축소할 수 있습니다. 동일한 흐름 보고서 내에서 다른 차원을 사용할 수도 있습니다.
