# 데이터 유무 문제 해결

Analysis Workspace에서 일부 프로젝트 설정은 0개의 행을 반환합니다. Reports &amp; Analytics에서 특정 보고서를 보면 다음 메시지가 표시됩니다.

**&quot;선택한 기준에 해당하는 데이터가 없습니다.&quot;**

보고서에 데이터가 표시되지 않는 이유를 결정하려면 다음 단계를 사용하십시오.

## 데이터가 있지만 필터링됩니다.

보고서 세트에는 데이터가 포함되어 있지만 보고서에 사용된 특정 구성 요소가 모든 데이터를 필터링합니다.

* **세그먼트**:세그먼트는 모든 데이터가 보고서에 표시되지 않도록 쉽게 방지할 수 있습니다. 모든 세그먼트 정의를 확인하고 모든 세그먼트를 제거하여 세그먼트가 데이터를 표시하지 않도록 하는지 확인합니다.
* **지표**:올바른 지표가 사용되는지 확인합니다. 지표를 [발생 횟수](/help/components/metrics/occurrences.md) 로 변경하여 이 지표가 데이터가 없는 보고서에 기여자가 아닌지 확인합니다.
* **날짜 범위**:날짜 범위가 과거 또는 미래의 어떤 때라도 너무 멀리 있으면 데이터를 반환하지 않습니다. 조직의 [데이터 보존 정책](data-retention.md)은(는) 백 데이터가 얼마나 오래 보관되는지를 결정합니다.
* **필터**:보고서에서 모든 검색 필터를 제거합니다.

## 데이터가 없습니다.

세그먼트가 없는 경우, 지표가 발생 횟수 이고, 날짜 범위가 현재 월이며 검색 필터가 없는 경우 다음을 확인하십시오.

* **보고서 세트를 확인합니다**.데이터가 포함된 보고서 세트에 있는지 확인하십시오. 많은 조직에는 일반적으로 많은 데이터를 포함하지 않는 새로운 보고서 세트, 테스트 또는 개발 보고서 세트가 있습니다.
* **지연**:최근 데이터를 보는 경우 데이터가 지연될 수 있습니다. 자세한 내용은 [지연](latency.md) 을 참조하십시오.
* **데이터 수집 유효성 검사**:보고서 세트가 추적해야 하는 웹 속성으로 이동하고  [Adobe 디버거를 엽니다](https://experienceleague.adobe.com/docs/debugger/using/experience-cloud-debugger.html). 페이지를 로드할 때 Analytics 이미지 요청이 있는지 확인합니다. 이미지 요청이 표시되는 경우 올바른 보고서 세트 ID를 사용하고, 올바른 [currencyCode](/help/implement/vars/config-vars/currencycode.md)를 포함하며, [타임스탬프 지원](/help/implement/vars/page-vars/timestamp.md)이 이미지 요청과 보고서 세트 간에 일치하는지 확인하십시오.
