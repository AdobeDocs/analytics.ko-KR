# Adobe Analytics에서 사용되는 용어

이 용어집을 사용하여 Adobe Analytics가 사용하는 많은 용어의 컨텍스트를 파악하십시오.

* **Activity Map:** 사이트에서 가장 많이 클릭한 영역을 보여주는 브라우저 플러그인입니다. 분석 사용 안내서에서 [Activity Map](../analyze/activity-map/activity-map.md) 를 참조하십시오.
* **Admin Console:** 참조할 수 있음:
   * Adobe Analytics의 보고서 세트 설정이 관리되는 기존 관리 도구입니다. 이전 버전의 Adobe Analytics 에서는 사용자 권한도 여기서 관리되었습니다. 관리 사용 안내서의 [관리 도구를](../admin/admin/c-admin-tools.md) 참조하십시오.
   * 제품 액세스가 프로비저닝되고 사용자 권한이 관리되는 Adobe Admin Console 관리 사용 안내서에서 [관리 콘솔을](../admin/admin-console/home.md) 참조하십시오.
* **할당:** 전환 변수가 방문 중에 둘 이상의 값을 발생하는 경우, 변수의 할당 설정에 따라 어떤 값이 보관됩니까? See [Conversion variables](../admin/admin/conversion-var-admin/conversion-var-admin.md) in the Admin user guide.
* **예외 항목:** 데이터에서 예상치 않은 트렌드를 자동으로 찾는 통계적 모델링 기능을 사용하여 감지했습니다. 이 모델은 지표를 분석하고 값의 하한, 상한 및 예상치 않은 범위를 파악합니다. 사용 안내서의 [예외 항목 탐지를](../analyze/analysis-workspace/virtual-analyst/c-anomaly-detection/anomaly-detection.md) 참조하십시오.
* **Appmeasurement:** 데이터를 수집하여 Adobe에 전송하는 데 사용되는 코드 라이브러리입니다. 구현 구현 안내서의 [홈 페이지를](../implement/home.md) 참조하십시오.
* **ASI 슬롯:** 더 이상 존재하지 않습니다. 이전 버전의 Adobe Analytics 에서는 ASI 슬롯이 세그먼트된 데이터를 볼 수 있는 임시 보고서 세트 컨테이너를 제공했습니다. 현재 버전의 Adobe Analytics 에서는 모든 보고서에 세그먼트를 즉시 적용할 수 있습니다.
* **분류:** 다른 차원의 컨텍스트 내에서 차원을 볼 수 있습니다. 사용자 [분석 안내서에서 차원](../analyze/analysis-workspace/components/dimensions/t-breakdown-fa.md) 분류를 참조하십시오.
* **바운스:** 단일 히트로 구성된 방문. 구성 요소 사용자 안내서의 [바운스를](../components/c-variables/c-metrics/metrics-bounces.md) 참조하십시오. 단일 액세스도 참조하십시오.
* **계산된 지표:** 보고에서 사용할 기존 지표, 통계 함수 및 공식을 조합할 수 있습니다. 구성 요소 사용자 안내서에서 [계산된 지표를](../components/c-calcmetrics/cm-overview.md) 참조하십시오.
* **캠페인:** 참조할 수 있음:
   * 추적 코드 차원을 채우는 캠페인 변수. 구현 구현 안내서에서 [페이지 변수를](../implement/js-implementation/c-variables/page-variables.md) 참조하십시오.
   * 추적 코드 차원의 기본 분류 모든 보고서 세트에 대해 자동으로 만들어집니다.
   * Adobe Campaign는 Adobe Experience Cloud에 포함되어 있습니다. [Adobe.com](https://www.adobe.com/marketing/campaign.html)에 대한 자세한 내용을 살펴보십시오.
* **채널:** 참조할 수 있음:
   * 사이트 섹션 차원을 채우는 채널 변수. 구현 구현 안내서에서 [페이지 변수를](../implement/js-implementation/c-variables/page-variables.md) 참조하십시오.
   * 마케팅 채널 - 사용자가 사이트에 도달하는 방법을 이해하는 데 도움이 되는 구성 요소입니다. 구성 요소 사용자 안내서의 [마케팅 채널을](../components/c-marketing-channels/c-overview.md) 참조하십시오.
* **분류:** 차원 값 그룹화를 허용하는 Adobe Analytics의 기능입니다. 구성 요소 사용자 안내서에서 [분류를](../components/c-classifications2/c-classifications.md) 참조하십시오.
* **Clickmap:** 더 이상 사용되지 않습니다. 사이트에서 가장 많이 클릭한 영역을 보여주는 레거시 브라우저 플러그인입니다. 이 도구는 Activity Map에 호의적으로 중단되었습니다.
* **클릭스트림 데이터 피드:** 데이터 피드를 참조하십시오.
* **집단:** 주어진 기간 동안 공통적인 특성을 공유하는 사람들의 그룹입니다. See [What is Cohort Analysis?](../analyze/analysis-workspace/visualizations/cohort-table/cohort-analysis.md) 를 참조하십시오.
* **컬렉션 서버:** 데이터 수집 서버를 참조하십시오.
* **컨텍스트 데이터 변수:** 처리 규칙에서만 사용되는 임시 변수. 처리 규칙이 전환 또는 트래픽 변수에 복사하지 않으면 컨텍스트 데이터 변수 값이 영구적으로 손실됩니다. 구현 구현 안내서에서 [컨텍스트 데이터 변수를](../implement/js-implementation/c-variables/context-data-variables.md) 참조하십시오.
* **전환 변수:** 동등하게 Evar로 알려져 있습니다. 사용자 지정 값을 저장하고 값이 만료될 때까지 변수 값을 유지합니다. See [Conversion variables](../components/c-variables/dimensionslist/reports-conversion.md) in the Components user guide.
* **상관 관계:** 더 이상 용어로 사용되지 않습니다. 차원 분류로 대체되었습니다. 이전 버전의 Adobe Analytics에서 상관 관계는 트래픽 변수를 분류하는 기능을 부여했습니다. 사용자 [분석 안내서에서 차원](../analyze/analysis-workspace/components/dimensions/t-breakdown-fa.md) 분류를 참조하십시오.
* **현재 데이터:** 아직 처리되지 않은 최근 수집된 데이터를 포함하도록 허용하는 일부 보고서의 옵션. 분석 사용 안내서에서 [현재 데이터를](../analyze/reports-analytics/current-data.md) 참조하십시오.
* **사용자 지정 링크:** 페이지 보기가 아닌 데이터를 포함하는 히트 유형. 구현 구현 안내서에서 [s. tl () 함수를](../implement/js-implementation/function-tl.md) 참조하십시오. 히트를 참조하십시오.
* **고객 속성:** 속성 데이터의 업로드를 허용하는 Experience Cloud 기능입니다. 핵심 서비스 사용 안내서에서 [고객 속성을](https://docs.adobe.com/content/help/en/core-services/interface/customer-attributes/attributes.html) 참조하십시오.
* **고객 지원 위임서:** Adobe 고객 지원 센터와 직접 상호 작용할 수 있는 지정된 사용자 Experience Cloud 기술 자료의 [고객 지원 담당자를](https://helpx.adobe.com/experience-cloud/supported-users.html) 참조하십시오.
* **데이터 수집 서버:** 데이터를 받고 처리하는 Adobe 소유 서버. 이미지 요청은 보고에서 사용하기 위해 Adobe의 데이터 수집 서버로 전송됩니다.
* **데이터 커넥터:** 제 3 자가 Adobe Analytics로 데이터 업로드를 자동화할 수 있는 완벽한 개발 솔루션입니다. 제 3 자 고객은 데이터 커넥터를 사용하여 Adobe Analytics에서 데이터를 보강할 수 있습니다. 대부분의 데이터 커넥터는 데이터 소스에 사용된 비슷한 워크플로우를 사용합니다. 가져오기 사용 안내서에서 데이터 커넥터를 참조하십시오.
* **데이터 피드:** 모든 히트를 행과 변수로 별도의 열로 나열하는 원시 데이터 내보내기입니다. Adobe Analytics 데이터를 타사 데이터베이스로 내보내는 데 가장 일반적으로 사용됩니다. 내보내기 사용 안내서에서 [데이터 피드를](../export/analytics-data-feed/c-getstarted/data-feed-overview.md) 참조하십시오.
* **Data Sources:** 사용자가 파일의 데이터를 Adobe Analytics로 업로드할 수 있습니다. 파일은 일반적으로 FTP 사이트에서 가져옵니다. 가져오기 사용자 안내서의 [데이터 소스를](../import/c-data-sources/datasrc-home.md) 참조하십시오.
* **데이터 웨어하우스:** 더 큰 보고서를 요청할 수 있는 Adobe Analytics의 기능입니다. 내보내기 사용 안내서에서 [데이터 웨어하우스를](../export/data-warehouse/data-warehouse.md) 참조하십시오.
* **Dimension:** 텍스트 같은 변수 값을 포함하는 구성 요소 유형입니다. 페이지 이름, 추적 코드 또는 참조 도메인이 포함됩니다. 지표는 일반적으로 해당 지표입니다.
* **다이내믹 태그 관리:** Adobe의 이전 태그 관리 솔루션 구현 구현 안내서에서 [DTM 구현 개요를](../implement/c-implement-with-dtm/dtm-implementation-overview.md) 참조하십시오. Adobe Experience Platform Launch를 대신 사용하는 것이 좋습니다.
* **이벤트 정리:** 중복 이벤트 수집을 방지하는 방법을 구현하는 프로세스. 구현 구현 안내서에서 [이벤트 정리를](../implement/js-implementation/event-serialization.md) 참조하십시오.
* **Evar:** 전환 변수를 참조하십시오.
* **이벤트:** 성공 이벤트를 참조하십시오.
* **Excelclient:** 더 이상 용어로 사용되지 않습니다. 리포트 빌더의 전임자 이름입니다.
* **만료:** 전환 변수의 컨텍스트에서는 값이 백엔드에서 지속되는 기간입니다. 이 지속성을 사용하면 이벤트가 이벤트 히트 전에 변수 값에 연결할 수 있습니다. See [Conversion variables](../admin/admin/conversion-var-admin/conversion-var-admin.md) in the Admin user guide.
* **흐름:** 사용자가 사이트에서 수행한 경로를 보여주는 시각화 작업 공간의 시각화 유형입니다. 사용자 분석 분석 안내서에서 [흐름 시각화를](../analyze/analysis-workspace/visualizations/c-flow/flow.md) 참조하십시오.
* **Genesis:** 더 이상 용어로 사용되지 않습니다. 데이터 커넥터의 이전 이름입니다.
* **글로벌 보고서 세트:** 여러 사이트의 히트를 수집하는 보고서 세트에 지정된 비공식 용어.
* **H 코드:** Appmeasurement의 선행 작업자입니다. 이전 버전의 Adobe Analytics 에서는 H .27 .5, H .26 등과 같은 "H 버전" 에 의해 코드 버전이 측정되었습니다.
* **히트:** Adobe 데이터 수집 서버로 전송된 단일 이미지 요청. 페이지 보기 횟수 및 사용자 지정 링크를 히트라고 합니다.
* **이미지 요청:** Adobe 데이터 수집 서버와 통신하는 데 사용되는 투명 1 x 1 픽셀 이미지입니다. 웹 사이트는 데이터가 들어 있는 긴 쿼리 문자열을 사용하여 이 보이지 않는 이미지를 요청합니다. Adobe는 보이지 않는 이미지를 반환하고 수신된 쿼리 문자열을 구문 분석합니다.
* **Insight:** 참조할 수 있음:
   * 데이터 워크벤치의 이전 이름입니다.
   * 사용자 지정 인사이트 - 사용자 지정 트래픽 변수에 대한 내역 이름입니다.
* **KPI:** 주요 성능 지표의 약어. 비즈니스가 사이트의 성과를 이해하는 데 도움이 되는 지표. 각 조직에는 비즈니스의 다양한 측면을 측정하는 다른 KPI가 있습니다. 구현 [구현 안내서에서 솔루션 디자인 문서](../implement/prepare/solution-design.md) 만들기를 참조하십시오.
* **지연:** 데이터가 수집될 때와 보고서에서 사용할 수 있는 시기 사이의 지연. 보고서 세트의 일반적인 지연은 30-90 분입니다. Technotes 사용자 안내서의 [지연을](../technotes/latency.md) 참조하십시오.
* **Launch:** Adobe의 현재 구현 솔루션인 Adobe Experience Platform Launch의 약칭입니다. Adobe Experience Platform 시작 사용자 안내서의 [개요를](https://docs.adobe.com/content/help/en/launch/using/overview.html) 참조하십시오.
* **목록 Prop:** 동일한 히트에서 여러 값을 지원하기 위해 일반적인 트래픽 변수를 전환하는 설정입니다. 설정이 활성화되면 모든 사용자 지정 트래픽 변수가 목록 Prop 이 될 수 있습니다. 구현 구현 안내서에서 [페이지 변수를](../implement/js-implementation/c-variables/page-variables.md) 참조하십시오.
* **목록 변수:** 전환 변수와 별도의 변수. 목록 변수는 동일한 히트에서 여러 값을 지원하며 변수 값은 전환 변수와 유사한 방문에서 유지됩니다. 조직에서 3 개의 목록 변수만 사용할 수 있습니다. 구현 구현 안내서에서 [페이지 변수를](../implement/js-implementation/c-variables/page-variables.md) 참조하십시오.
* **로그인 회사:** 조직에서 사용하는 보고서 세트 모음입니다. 일부 조직에는 조직의 다른 부분에 적용되는 여러 로그인 회사가 있습니다.
* **마케팅 채널:** 히트를 사이트에 도달했을 때 분류를 분류하는 Adobe Analytics의 기능입니다. 히트를 분류하는 데 사용된 로직은 마케팅 채널 처리 규칙을 사용하여 사용자 지정할 수 있습니다. 구성 요소 [사용 안내서에서 마케팅 채널](../components/c-marketing-channels/c-getting-started-mchannel.md) 시작하기를 참조하십시오.
* **지표:** 양적 데이터를 포함하는 구성 요소 유형입니다. 지표 값에는 일반적으로 페이지 보기 횟수, 방문 횟수 및 매출액과 같은 숫자가 포함됩니다. 차원은 일반적으로 그 반대입니다.
* **다중 세트 태깅:** 여러 보고서 세트에 동일한 히트를 전송하는 방법입니다. 가상 보고서 세트를 도입하면 이 방법이 더 이상 필요하지 않습니다. 대부분의 다중 세트 태깅 노력은 글로벌 보고서 세트를 수용하는 데 도움이 됩니다.
* **표준화:** 모든 지표를 가져와 동일한 비율을 적용하여 트렌드를 보다 쉽게 비교할 수 있는 시각화를 구성하는 방법입니다.
* **발생 횟수:** 차원 값이 설정되었거나 지속된 히트 수를 보여주는 지표 유형입니다. 구성 요소 사용자 안내서의 [발생을](../components/c-variables/c-metrics/metrics-occurrences.md) 참조하십시오.
* **Omniture:** 더 이상 용어로 사용되지 않습니다. 2009 년 Adobe에 인수되기 전에 Adobe Analytics를 소유한 조직
* **경로 지정:** 흐름을 참조하십시오.
