# 단일 액세스

**[!UICONTROL 단일 액세스]** [지표](overview.md)은(는) 적용 가능한 보고 차원에 전체 방문에 대한 단일 값만 포함된 방문의 수를 보여줍니다. [[!UICONTROL 단일 페이지 방문]](single-page-visits.md)의 광범위한 차원별 버전입니다. 이 지표는 방문 중에 한 번만 설정된 경우 차원의 값을 보려는 차원 컨텍스트에서 유용합니다.

## 이 지표의 계산 방법

이 지표의 정의는 [[!UICONTROL 반복 인스턴스 계산]](/help/analyze/analysis-workspace/build-workspace-project/create-projects.md#project-info-settings)의 프로젝트 설정에 따라 다릅니다.

* **반복 인스턴스 계산 사용**: 차원에 하나의 방문 값이 있는 방문을 계산합니다. 차원이 지속되면 더 이상 단일 액세스로 분류되지 않습니다.
* **반복 인스턴스 계산 사용 안 함**: 차원에 하나의 고유 값이 포함된 방문을 계산합니다. 차원 항목을 동일한 값으로 여러 번 설정하거나 지속시켜 단일 액세스로 계속 계산되도록 할 수 있습니다.

&#39;[!UICONTROL 반복 인스턴스 계산]&#39;에 관계없이, 차원이 두 번째 고유 값으로 바뀌는 경우 이 방문은 더 이상 단일 액세스로 분류되지 않습니다. 링크 추적 호출에 차원이 설정된 경우 이 계산에 포함됩니다.

## &#39;[!UICONTROL 단일 액세스]&#39;과(와) &#39;[!UICONTROL 단일 페이지 방문]&#39;의 차이점

[[!UICONTROL 페이지]](../dimensions/page.md) 차원의 컨텍스트에서 &#39;[!UICONTROL 단일 액세스]&#39;과(와) &#39;[!UICONTROL 단일 페이지 방문]&#39;은(는) &#39;[!UICONTROL 반복 인스턴스 계산]&#39; 프로젝트 설정에 관계없이 항상 동일합니다. 그 차이는 다른 차원을 사용할 때 나타납니다.

* **[!UICONTROL 단일 액세스]**: 단일 히트에 대해 주어진 차원 항목이 있는 방문의 수를 표시합니다. 프로젝트에서 사용하는 차원의 컨텍스트와 관련되어 있습니다.
* **[!UICONTROL 단일 페이지 방문]**: 단일 히트에 대해 &#39;[!UICONTROL 페이지]&#39; 차원이 있는 방문 횟수를 표시합니다. 보고서에서 다른 차원을 사용하더라도 여전히 하나의 고유한 &#39;[!UICONTROL 페이지]&#39; 차원 항목을 포함하는 방문을 계산합니다.

[[!UICONTROL 반복 인스턴스 계산]](/help/analyze/analysis-workspace/build-workspace-project/create-projects.md#project-info-settings)을 사용하지 않도록 설정하면 지표 정의가 약간 변경됩니다.

* **단일 액세스**: 전체 방문에 대해 주어진 차원 항목이 변경되지 않은 방문의 수를 표시합니다.
* **단일 페이지 방문**: 전체 방문에 대해 &#39;[!UICONTROL 페이지]&#39; 차원이 변경되지 않은 방문의 수를 표시합니다.

예를 들어 다음 두 번의 히트 방문 예를 고려해 보십시오. 보고서의 차원은 [[!UICONTROL 사이트 섹션]](../dimensions/site-section.md)이며 &#39;[!UICONTROL 반복 인스턴스 계산]&#39;을(를) 사용할 수 없습니다.

* 방문자가 두 페이지를 방문하지만 둘 다 동일한 사이트 섹션에 있습니다. 사이트 섹션이 방문 내내 동일하게 유지되므로 단일 액세스로 계산됩니다. 이 방문은 둘 이상의 고유한 [!UICONTROL 페이지] 차원 항목을 포함하므로 단일 페이지 방문으로 계산되지 않습니다.
* 방문자가 서로 다른 사이트 섹션의 두 페이지를 방문하지만 두 페이지의 이름이 동일합니다. 두 개의 고유한 사이트 섹션 값이 있었으므로 방문은 단일 액세스로 계산되지 않습니다. 하나의 고유한 [!UICONTROL 페이지] 차원 항목이 있었으므로 단일 페이지 방문으로 계산됩니다.
