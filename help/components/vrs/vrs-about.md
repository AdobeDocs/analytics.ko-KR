# 가상 보고서 세트 개요

가상 보고서 세트는 Adobe Analytics 데이터를 세그먼트로 분할하므로 각 세그먼트에 대한 액세스를 제어할 수 있습니다.

많은 고객에게 글로벌 보고서 세트로 유입되는 데이터가 있지만 더 작은 보고서 세트로 유입되는 데이터도 있습니다. 변수를 여러 보고서 세트로 설정하고 데이터를 두 개 이상의 보고서 세트로 보냅니다. 이를 *다중 세트 태그 지정* 또는 *기본/상위 보고서 세트*&#x200B;라고 합니다.

예를 들어 모든 데이터를 하나의 보고서 세트에 수집할 수도 있지만 회사 직원 내 다른 사람이 모든 데이터가 아닌 일부 데이터에 액세스할 수 있도록 보조 보고서 세트를 설정할 수도 있습니다. 데이터는 지역별로 나뉠 수 있습니다. 국가마다 다른 웹 사이트가 있을 수 있습니다. 다른 예는 규모가 큰 회사에 속하는 특정 브랜드일 수 있지만, 각 브랜드에는 고유한 마케팅 팀이 있습니다.

*가상 보고서 세트*&#x200B;를 사용하면 여러 보고서 세트 대신 세그먼트를 사용하여 이 분기 개념을 재현할 수 있습니다. 데이터는 하나의 보고서 세트로 전송된 다음 세그먼트별로 나누어집니다. 여러 브랜드 예를 사용하면 항목이 속한 브랜드에 대한 prop을 설정할 수 있습니다. 세그먼트를 사용하여 각 prop에 할당된 항목에 대해 보고할 수 있습니다. 이러한 각 세그먼트는 자체 보기가 되어 새 보고서 세트를 효과적으로 만듭니다. 해당 세그먼트에만 특별히 데이터를 보내지 않고, 글로벌 보고서 세트에만 보내지만, 보고서에는 다른 보고서 세트처럼 작동합니다.

가상 보고서 세트는 eVar 설정, 처리 규칙, 분류 등과 같은 기본 보고서 세트의 대부분의 서비스 수준을 상속합니다. 다음 설정은 상속되지 않습니다.

* 보고서 세트 ID (RSID)
* 보고서 세트 이름
* 권한 그룹 (가상 보고서 세트를 고유한 권한 그룹에 할당할 수 있음)

## 가상 보고서 세트의 장점 {#section_3420422FE6DF46EAB151FD9442AAFDC4}

보조 서버 호출에 대한 비용을 고객이 지불하므로 이러한 호출을 제거하면 상당한 비용 절감 효과를 얻을 수 있습니다. 가상 보고서 세트도 완전히 소급됩니다. 글로벌 보고서 세트에 이미 데이터가 포함되어 있는 경우 관련 데이터가 자동으로 새 가상 보고서 세트에 포함됩니다. 새 보조 보고서 세트는 생성된 후에만 데이터 수집을 시작하므로 내역 데이터를 포함하지 않습니다. Analytics를 구현할 때 글로벌 보고서 세트 및 각 보조 보고서 세트에 대한 구현을 만들지 않고도 데이터를 하나의 보고서 세트에만 전송하면 됩니다.

가상 보고서 세트 도움말:

* 모든 사이트/도메인에서 RSID(보고서 세트 ID)를 사용할 수 있도록 하여 구현을 간소화합니다. 모든 데이터를 단일 보고서 세트에 포함하면 차세대 Adobe Analytics으로 나아갈 때 고객 분석을 사용할 수 있습니다.
* 조직의 비즈니스 사용자는 항상 자신과 관련된 데이터 세그먼트만 봅니다.
* 관리 사용자가 구현 후 데이터 액세스를 보다 쉽고 세부적으로 제어할 수 있도록 하여 보안을 개선합니다.
* 인물 지표
* 데이터에 대한 단일 고객 보기 (향후)
* 데이터를 세그먼트화할 수 있는 무제한 가상 보고서 세트 생성 기능

## 가상 보고서 세트의 한계 {#section_F22A6DEBDC9848429E446F4CC2C4EEDE}

가상 보고서 세트에는 다음과 같은 한계가 있습니다.

* 세그먼트의 모든 제한 사항이 가상 보고서 세트에 적용됩니다

  가상 보고서 세트는 보고서 세트에 적용되는 세그먼트에 불과합니다. 각 보고서 세트에는 자체 Data Warehouse 및 자체 Data Feed가 있으므로 여러 보고서 세트를 사용하면 세그먼트에서 제공하지 않는 몇 가지 이점이 있습니다.
* 실시간 보고서
* 전체 보고서 세트에서처럼 설정 및 변수 이름을 사용자 정의할 수 없습니다

## 가상 보고서 세트와 다중 세트 태그 지정 결합 {#section_026FA3FCD7314DD18220E73EC5702AFF}

경우에 따라 가상 보고서 세트와 다중 세트 태깅을 모두 사용하면 이점이 있습니다.

예를 들어 retailer은 각 브랜드에 대한 보고서 세트를 사용하고 각 브랜드에 대한 가상 보고서 세트를 사용하여 데이터를 지역별로 분류할 수 있습니다. 마찬가지로 체육 단체는 각 팀에 대한 보고서 세트를 사용한 다음 가상 보고서 세트를 사용하여 팀 지역의 팬과 지역 외부의 팬을 나눌 수 있습니다.
