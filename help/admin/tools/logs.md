# 로그

사용자가 로그인하는 시점, 사용자의 사용, 액세스, 보고서 세트 및 관리 변경을 확인하는 데 도움이 되는 로그 파일입니다.

**[!UICONTROL Analytics]** > **[!UICONTROL 관리자]** > **[!UICONTROL 모든 관리자]** > **[!UICONTROL 로그]**

## 관리 로그 {#section_8ADE8A7204A8401C968ABC20AECA381D}

관리 로그는 관리자가 관리 도구에서 수행한 모든 변경을 보고합니다. 이 로그는 세 가지 로그 중 하나의 사용자 정의 보고서에 대한 게이트웨이를 제공합니다. 지정된 날짜 범위 동안 선택한 기준과 일치하는 이벤트를 검색할 수 있습니다.

## 사용 및 액세스 로그 {#section_6FBAF92D9EA244809C45A78A2F0A7232}

[!UICONTROL 사용 및 액세스 로그]를 통해 사용자 계정 수준에서 보고서 사용을 평가할 수 있습니다. 예를 들어 Analysis Workspace에서 열기, 만들기, 업데이트, 공유 해제 및 삭제 작업을 추적합니다. 이를 통해 Workspace를 사용하는 사용자와 빈도를 더욱 잘 파악할 수 있습니다.

| 요소 | 설명 |
|---|---|
| 날짜 범위 | 날짜 범위 필터를 지정합니다. 날짜를 YYYY-MM-DD 형식으로 수동으로 입력하거나 달력 아이콘을 클릭하여 날짜를 선택할 수 있습니다. |
| 로그인 | 사용자 이름으로 로그를 필터링합니다. |
| IP | IP 주소로 로그를 필터링합니다. |
| 보고서 세트 | 특정 보고서 세트 ID로 로그를 필터링합니다. |
| 이벤트 유형 | 이벤트 유형별로 로그를 필터링합니다. 드롭다운 목록에서 이벤트 유형을 선택합니다. 아래 전체 이벤트 유형 목록을 참조하십시오. |
| 이벤트 | 이벤트 설명의 단어 또는 구문으로 로그를 필터링합니다. |
| 보고서 다운로드 | [!UICONTROL 사용 및 액세스 로그]의 내용을 탭으로 구분된 파일로 내보냅니다. |

## 보고서 세트 변경 로그 {#section_3864966639414BBEA871F4D0352F56B6}

보고서 세트 변경 로그에는 관리자 외부에서 보고서 세트에 수행된 변경 사항이 표시됩니다.

[!UICONTROL 관리 도구] 외부에서 보고서 세트를 수정할 수 있는 도구는 다음과 같습니다.

* 웹 브라우저에서 수행한 분류 업로드(FTP를 통해 수행한 분류 업로드는 변경 로그에 포함되지 않음)
* 이전 버전에서 변경된 내용.
* 계정 담당자 또는 고객 지원 센터에서 내부 도구를 사용하여 변경한 사항

| 요소 | 설명 |
|---|---|
| 날짜 범위 | 날짜 범위 필터를 지정합니다. 날짜를 YYYY-MM-DD 형식으로 수동으로 입력하거나 달력 아이콘을 클릭하여 날짜를 선택할 수 있습니다. |
| 회사 | 회사 이름별로 로그를 필터링합니다. |
| 로그인 | 사용자 이름으로 로그를 필터링합니다. |
| IP | IP 주소로 로그를 필터링합니다. |
| 이벤트 | 이벤트 설명의 단어 또는 구문으로 로그를 필터링합니다. |
| 보고서 다운로드 | [!UICONTROL 사용 및 액세스 로그]의 내용을 탭으로 구분된 파일로 내보냅니다. |
