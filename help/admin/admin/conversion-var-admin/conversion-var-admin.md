# 전환 변수(eVar)

사용자 지정 인사이트 전환 변수(또는 eVar)는 사이트의 선택된 웹 페이지의 Adobe 코드에 삽입됩니다. 주요 목적은 사용자 지정 마케팅 보고서에서 전환 성공 지표를 세그먼트화하는 것입니다. eVar는 방문 기반이며 쿠키와 유사한 기능을 수행할 수 있습니다. eVar 변수에 전달된 값은 미리 결정된 기간 동안 사용자를 따릅니다.

eVar가 방문자에 대한 값으로 설정되면 Adobe는 값이 만료될 때까지 해당 값을 자동으로 기억합니다. eVar 값이 활성 상태일 때 방문자가 발견하는 성공 이벤트는 eVar 값으로 계산됩니다.

eVar는 다음과 같은 원인과 효과를 측정하는 데 가장 적합합니다.

* 매출에 영향을 준 내부 캠페인
* 등록으로 이어지는 배너 광고
* 주문 전 내부 검색이 사용된 횟수

트래픽 측정 또는 경로 지정을 원하는 경우 트래픽 변수를 사용하는 것이 좋습니다.

> [!NOTE] 단일 값만 이미지 요청 시 eVar에 저장할 수 있습니다. eVar 값에 여러 값을 사용하려는 경우에는 [목록 변수(list vars)](https://marketing.adobe.com/resources/help/ko_KR/sc/implement/listN.html)를 구현하는 것이 좋습니다.

**만료**

`eVars`은 지정되는 일정 시간 후 만료되며, eVar가 만료되면 더 이상 성공 이벤트에 대한 크레딧을 받지 않습니다. eVar는 성공 이벤트에 대해 만료되도록 구성할 수도 있습니다. 예를 들어, 방문이 끝날 때 만료되는 내부 판촉 행사가 있는 경우 내부 판촉 행사는 해당 판촉 행사가 활성화된 방문 동안 발생하는 구매 또는 등록에 대해서만 크레딧을 받습니다.

eVar를 만료하는 두 가지 방법이 있습니다.

* eVar가 지정된 기간 또는 이벤트 후에 만료되도록 설정할 수 있습니다.
* eVar를 재설정하여 강제 만료 기능을 사용할 수 있습니다. 이 기능은 변수를 재활용할 때 유용합니다.

예를 들어 eVar 만료를 30일에서 90일로 변경하면 수집된 eVar 값은 새 만료 세트의 기간(이 경우 90일)에 계속 유지됩니다. 시스템은 현재 만료 설정 및 수집된 eVar 값의 마지막 세트 타임스탬프를 보고 만료를 결정합니다. 옵션만 값을 **[!UICONTROL Reset]** 만료하고 즉시 만료됩니다.

다른 예:eVar를 내부 판촉 행사를 반영하기 위해 5월에 사용하고 6월에 내부 검색 키워드를 캡처하는 데 사용하는 경우 6월 1일에 변수를 강제로 만료하거나 재설정해야 합니다. 이렇게 하면 내부 판촉 행사 값을 6월의 보고서에서 빼는 데 유용합니다.

**대/소문자 구분**

eVar는 대/소문자를 구분하지 않지만 첫 번째 발생 시 대/소문자가 표시됩니다. 예를 들어, eVar1의 첫 번째 인스턴스가 &quot;로그인&quot;으로 설정되었지만 그 이후의 모든 인스턴스가 &quot;로그인&quot;으로 전달되면 보고서는 항상 &quot;로그인&quot;을 eVar 값으로 표시합니다.

**카운터**

eVar는 문자열 값을 유지하는 데 가장 많이 사용되는 반면, 카운터로 작동하도록 구성할 수도 있습니다. eVar는 이벤트 전에 사용자가 수행한 작업 수를 카운트하려는 경우 카운터로 유용합니다. 예를 들어 eVar를 사용하여 구매 전에 내부 검색 수를 캡처할 수 있습니다. 방문자가 검색할 때마다 eVar에 &#39;+1&#39; 값이 있어야 합니다. 방문자가 구매 전에 4번 검색하는 경우 각 총 계수에 대한 인스턴스가 표시됩니다.1.00, 2.00, 3.00 및 4.00그러나 구매 이벤트(주문 및 매출 지표)에 대한 크레딧은 4.00만 받습니다. eVar 카운터의 값으로 양수만 사용할 수 있습니다.
